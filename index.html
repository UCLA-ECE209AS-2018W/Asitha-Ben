Blockchain Framework for Internet of Things

Motivations

P2P Vulnerabilities

The prime motivation for this project was addressing the security risk inherent to P2P networks that a single node being compromised, in order to alter data or for imitation by a malicious actor, can wreak havoc on the rest of the network. At the same time our solution addresses malicious actors attempting to join a private network as if they were an insider. For instance, in a home security system, if an outside IoT device managed to convince a single critical network node that it had permission to unlock the front door, all security on all other nodes can do nothing to prevent it. By adding the blockchain element to the system, a network with heavy amounts of redundancy and immutability, as well as a secure consensus system, can ensure the performance of the security layers as long as a majority of the nodes are uncompromised.

IoT Vulnerabilities

The risks that come along with the use of a P2P network are made even more likely through the use of IoT and embedded devices, which are often unsecured and prime targets for attack. Many devices are connected to the network and out of sight of any supervision, with unmitigated physical access points such as usb and JTAG ports, as well as weakly protected network access points. These vulnerabilities are exacerbated by holes in the existing security layer on the device, such as weak or nonexistent authentication and encryption, and often old firmware that has not been patched for discovered exploits.

Project Summary

Our goal for this project is to address some of these concerns by building a blockchain framework (based off Flowchain) for an IoT network, while ensuring security and stability.

In order to implement this system, we dove into the inner components of Flowchain to explore how it works, and added a few features to increase the functionality of the framework and ensure security within the network. We stressed keeping the system simple and lightweight enough to reliably run on embedded devices, while also adding significant layers of verification and authentication. Our main contributions to this project were reformatting the network structure itself in order to create a private IoT blockchain network that is capable of securely managing membership in a home or industrial setting.
